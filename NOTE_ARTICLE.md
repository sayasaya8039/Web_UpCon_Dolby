# YouTubeの音質が劇的に変わる！無料で使える「Web UpCon Dolby」を作りました

## はじめに

「YouTubeやAmazon Musicの音質、なんか物足りないな...」

そう感じたことはありませんか？

実は、ストリーミングサービスの音声は**データ量削減のために圧縮**されており、本来の音楽が持つ高音域や空間の広がりが失われています。

そこで、**ブラウザ上でリアルタイムに音質を補完・拡張するChrome拡張機能**を開発しました。

その名も **「Web UpCon Dolby」**

---

## この拡張機能でできること

### 1. ハイレゾ相当へのアップサンプリング

通常の配信音源（44.1kHz/48kHz）を、**最大192kHzまでアップサンプリング**。

圧縮で失われた高周波成分を独自のアルゴリズムで補完し、CDを超えるハイレゾ相当の音質を実現します。

| 元の音源 | 変換後 |
|---------|--------|
| 44.1kHz | → 96kHz / 192kHz |
| ~16kHz | → ~48kHz（高域拡張） |

### 2. 疑似Dolby Atmos / 7.1chサラウンド

普通のイヤホン・ヘッドホンで、まるで**映画館のような立体音響**を体験できます。

- **ステレオワイド**: 音の広がりを自然に拡張
- **7.1chサラウンド**: 後方からの音も再現
- **Dolby Atmos風**: 頭上からの音も含めた3D空間を演出

### 3. ワンクリックで切り替え可能

難しい設定は不要。**プリセットを選ぶだけ**で最適な音質に。

- **Music**: 音楽視聴に最適化（ハイレゾ＋空間オーディオ）
- **Movie**: 映画・動画向け（サラウンド重視）
- **Gaming**: ゲーム向け（低遅延＋定位感重視）

## インストール方法

### Step 1: 拡張機能をダウンロード

GitHubから最新版をダウンロードします。

**[→ GitHubリリースページ](https://github.com/sayasaya8039/Web_UpCon_Dolby)**

「Code」→「Download ZIP」でダウンロードし、解凍してください。

### Step 2: Chromeに読み込む

1. Chromeで `chrome://extensions` を開く
2. 右上の「**デベロッパーモード**」をONに
3. 「**パッケージ化されていない拡張機能を読み込む**」をクリック
4. 解凍したフォルダ内の「**Web_UpCon_Dolby**」フォルダを選択

### Step 3: 完了！

ツールバーに音符アイコンが表示されたら準備完了です。

## 使い方

### 基本操作

1. YouTubeなどで動画・音楽を再生
2. ツールバーの拡張機能アイコンをクリック
3. メインスイッチを**ON**に
4. お好みのプリセットを選択

### おすすめ設定

**音楽をじっくり聴きたい時**
```
プリセット: Music
ハイレゾ: ON
空間オーディオ: ON
サンプルレート: 96kHz
```

**映画を迫力ある音で楽しみたい時**
```
プリセット: Movie
ハイレゾ: ON
空間オーディオ: ON（7.1chサラウンド）
```

**ゲーム配信を見る時**
```
プリセット: Gaming
ハイレゾ: OFF（低遅延優先）
空間オーディオ: ON（Atmos）
```

## 技術的なこだわり

### リアルタイム処理

Web Audio APIとAudioWorkletを活用し、**遅延を最小限に抑えたリアルタイム処理**を実現。動画と音声のズレを感じることなく高音質を楽しめます。

### 高品質Sinc補間

アップサンプリングには、プロの音楽制作でも使われる**Sinc補間アルゴリズム**を採用。単純な線形補間とは違う、滑らかで自然な高音域を生成します。

### HRTF空間処理

人間の耳の構造（頭部伝達関数 = HRTF）をシミュレートし、**普通のステレオヘッドホンで立体音響**を実現しています。

### GPUアクセラレーション対応

WebGPU対応環境では、GPU処理による高速化も可能。CPU負荷を抑えながら高品質な音声処理ができます。

## よくある質問

### Q: 本当に音質は変わりますか？

A: はい。特に以下の点で違いを感じられます：
- 高音域の伸び（シンバル、ボーカルの息づかいなど）
- 音の広がりと奥行き
- 低音の締まり

### Q: 動画と音声がズレませんか？

A: AudioWorkletによる低遅延処理を行っているため、通常の視聴では気にならないレベル（数ミリ秒）に抑えています。

### Q: スマホでも使えますか？

A: 現時点ではPC版Chrome/Edge専用です。将来的にはFirefox対応も検討中です。

### Q: バッテリー消費は増えますか？

A: 音声処理を行うため、多少の増加はあります。気になる場合は「Gaming」プリセットで軽量な処理に切り替えてください。

## 開発の経緯

普段からYouTubeで音楽を聴くことが多いのですが、どうしても「圧縮音源特有の物足りなさ」が気になっていました。

高価なDACやアンプを買わなくても、**ソフトウェア処理で音質を改善できないか？** という思いから、この拡張機能の開発をスタート。

Web Audio APIの可能性を追求し、ブラウザ上でリアルタイムに動作する高音質化エンジンを実現しました。
